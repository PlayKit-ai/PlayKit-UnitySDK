# Quick Start Guide

This guide will help you get started with the Developerworks Unity SDK quickly. Follow these steps to create your first integration.

## Prerequisites

Before you begin, make sure you have:

- Completed the [Installation](/docs/installation) process
- Set up your API credentials
- A Unity project ready to work with

## Step 1: Initialize the SDK

First, initialize the SDK in your main game scene:

```csharp
using UnityEngine;
using Developerworks.SDK;

public class GameManager : MonoBehaviour
{
    void Start()
    {
        // Initialize the SDK
        DeveloperworksSDK.Initialize(new SDKConfig
        {
            ApiKey = "your-api-key",
            ApiSecret = "your-api-secret",
            Environment = SDKEnvironment.Production
        });

        Debug.Log("SDK initialized successfully!");
    }
}
```

<Callout type="info">
  For production apps, load credentials from a secure configuration file instead of hardcoding them.
</Callout>

## Step 2: Authenticate a User

Set up user authentication:

```csharp
using Developerworks.SDK.Auth;

public class AuthManager : MonoBehaviour
{
    async void AuthenticateUser()
    {
        try
        {
            // Authenticate with username and password
            var result = await DeveloperworksAuth.SignIn(
                username: "user@example.com",
                password: "secure-password"
            );

            if (result.Success)
            {
                Debug.Log($"Welcome, {result.User.DisplayName}!");
                OnAuthenticationSuccess(result.User);
            }
            else
            {
                Debug.LogError($"Authentication failed: {result.Error}");
            }
        }
        catch (System.Exception ex)
        {
            Debug.LogError($"Authentication error: {ex.Message}");
        }
    }

    void OnAuthenticationSuccess(User user)
    {
        // Handle successful authentication
        // e.g., load user profile, navigate to main menu, etc.
    }
}
```

## Step 3: Make Your First API Call

Once authenticated, you can start making API calls:

```csharp
using Developerworks.SDK.API;

public class DataManager : MonoBehaviour
{
    async void FetchUserData()
    {
        try
        {
            // Get current user's data
            var userData = await DeveloperworksAPI.GetUserData();

            Debug.Log($"User Level: {userData.Level}");
            Debug.Log($"Total Points: {userData.Points}");

            // Update user data
            await DeveloperworksAPI.UpdateUserData(new UserDataUpdate
            {
                Level = userData.Level + 1,
                Points = userData.Points + 100
            });

            Debug.Log("User data updated successfully!");
        }
        catch (APIException ex)
        {
            Debug.LogError($"API error: {ex.Message}");
        }
    }
}
```

## Step 4: Handle Events

Subscribe to SDK events to respond to important changes:

```csharp
public class EventHandler : MonoBehaviour
{
    void OnEnable()
    {
        // Subscribe to events
        DeveloperworksSDK.OnConnectionStateChanged += OnConnectionChanged;
        DeveloperworksSDK.OnDataSynced += OnDataSynced;
        DeveloperworksAuth.OnAuthStateChanged += OnAuthChanged;
    }

    void OnDisable()
    {
        // Unsubscribe from events
        DeveloperworksSDK.OnConnectionStateChanged -= OnConnectionChanged;
        DeveloperworksSDK.OnDataSynced -= OnDataSynced;
        DeveloperworksAuth.OnAuthStateChanged -= OnAuthChanged;
    }

    void OnConnectionChanged(ConnectionState state)
    {
        Debug.Log($"Connection state: {state}");
    }

    void OnDataSynced()
    {
        Debug.Log("Data synchronized with server");
    }

    void OnAuthChanged(AuthState state)
    {
        if (state == AuthState.SignedOut)
        {
            Debug.Log("User signed out");
            // Handle sign out (e.g., return to login screen)
        }
    }
}
```

## Complete Example

Here's a complete example bringing it all together:

```csharp
using UnityEngine;
using Developerworks.SDK;
using Developerworks.SDK.Auth;
using Developerworks.SDK.API;

public class CompleteExample : MonoBehaviour
{
    void Start()
    {
        InitializeAndAuthenticate();
    }

    async void InitializeAndAuthenticate()
    {
        // 1. Initialize SDK
        DeveloperworksSDK.Initialize(new SDKConfig
        {
            ApiKey = LoadApiKey(),
            ApiSecret = LoadApiSecret(),
            Environment = SDKEnvironment.Production
        });

        // 2. Subscribe to events
        DeveloperworksAuth.OnAuthStateChanged += OnAuthStateChanged;

        // 3. Authenticate user
        var authResult = await DeveloperworksAuth.SignIn(
            username: "user@example.com",
            password: "secure-password"
        );

        if (authResult.Success)
        {
            // 4. Fetch user data
            var userData = await DeveloperworksAPI.GetUserData();
            Debug.Log($"Welcome back, {userData.DisplayName}!");
        }
    }

    void OnAuthStateChanged(AuthState state)
    {
        Debug.Log($"Auth state changed to: {state}");
    }

    string LoadApiKey()
    {
        // Load from secure storage
        return PlayerPrefs.GetString("ApiKey");
    }

    string LoadApiSecret()
    {
        // Load from secure storage
        return PlayerPrefs.GetString("ApiSecret");
    }

    void OnDestroy()
    {
        // Clean up
        DeveloperworksAuth.OnAuthStateChanged -= OnAuthStateChanged;
    }
}
```

## Best Practices

As you build with the SDK, keep these best practices in mind:

- **Security**: Never hardcode API credentials in your scripts
- **Error Handling**: Always use try-catch blocks for async operations
- **Event Management**: Remember to unsubscribe from events to prevent memory leaks
- **Async/Await**: Use async/await for all SDK API calls
- **Testing**: Test authentication and API calls in both Editor and build environments

## Common Issues

### "SDK not initialized" error

Make sure to call `DeveloperworksSDK.Initialize()` before any other SDK methods.

### Authentication fails

- Verify your API credentials are correct
- Check your internet connection
- Ensure the user's credentials are valid

### API calls timeout

- Check your network connection
- Verify the API endpoint is accessible
- Review your firewall settings

For more help, consult the [API Reference](/docs/api-reference) or contact support.
